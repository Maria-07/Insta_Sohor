Some Questions & Answers

JavaScript is an interpreter language that checks the entire code line by line. JavaScript always has to run in the browser, when we try to run a JavaScript file in the browser it's run by the browser's own JS engine. There are different JS engines for different browsers such as the V8 engine for Chrome, SpiderMonkey for Firefox, Chakra for Internet explorer, etc.

Firstly the javascript file parses the code inside the parser which is inside the engine, which checks the entire code line by line. which Inform if there is any problem with the code. The parser then produces the AST (Abstract Syntax Tree) which converts the code into machine code. Then the actual JS code runs and shows the output on the screen.

javaScript is a combination of asynchronous and synchronous language. It's primarily single-threaded. During the event Loop situation, it behaves asynchronously.

allocation and the call stack part is used for stack all those functions which are called during code run time. Stack always works as a LIFO(last in first out) so when multiple functions call it's synchronously executed. but when there happened any setTime Function, there occurs an event loop. the call stack sent the set time function to the Web API to execute and at the same time, it executed other functions synchronously. when the web API completes the execution it sent the function to the callback queue.

where Queue is FIFO(First in First Out) . So when the call stack finished execution of all functions here the event loop occurs. the event loop connects the callback queue with the call stack. JavaScript does multiple works via Event Loop on the asynchrony way.

Local storage and session storage are a property of the DevTool application in javaScript. which works for some storage purposes.

the main differences between session and local storage are the value which we set in local storage stored until it is Cleared or Deleted. At the same time, the values set in the session storage reset or delete during all the reload or refresh times.

there are also some data limitations among them. it says the 5MB limit for local storage was 5-10MB in session storage. Unlike cookies, local and session storage data are not transferred on each HTTP request.
